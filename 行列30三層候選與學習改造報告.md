# 行列30三層候選與學習改造報告

日期：2026-05-12

## 一、這次改造要解決什麼

這次改造的目標，是讓行列30在 Rime 裡同時滿足三件事：

1. A：單字碼表固定排序，不動態調頻。
2. B：原生詞組碼表固定排序，不動態調頻。
3. C：連續打單字形成詞組或句子時，仍然可以學習你的使用習慣。

改造前，這三件事主要混在同一個 `table_translator` 裡處理。這就像三種不同業務都擠在同一個櫃台辦理：

- 單字候選也會受用戶詞頻影響。
- 原生詞組候選也會受用戶詞頻影響。
- 語句連打學習雖然能用，但會和前兩者互相牽動。

所以最初的矛盾是：一旦開啟用戶詞典，A 和 B 也跟著調頻；一旦關閉用戶詞典，C 又失去學習能力。

這次最終採用的思路，是把原本混在一起的輸入來源拆成三條獨立管線。

## 二、改造後的三條管線

現在的結構可以理解成三個櫃台，各自負責不同任務。

### A：固定單字櫃台

負責來源：

- `array30_char.dict.yaml`

用途：

- 專門輸出單字。
- 不使用用戶詞典。
- 不動態調頻。
- 權重最高，確保單字候選先按碼表原序出現。

例如 `nor`：

```text
產    nor
性    nor
```

因為這條管線關閉了用戶詞典，所以即使你連續選很多次「性」，它也不會跑到「產」前面。

核心設定在 `array30.custom.yaml`：

```yaml
character:
  dictionary: array30_char
  enable_completion: false
  enable_sentence: false
  enable_user_dict: false
  initial_quality: 1000
```

這裡的重點是：

- `dictionary: array30_char`：只看單字詞典。
- `enable_user_dict: false`：不參與用戶調頻。
- `initial_quality: 1000`：給它很高的初始權重，讓固定單字優先。

### B：固定原生詞組櫃台

負責來源：

- `array30_phrases.dict.yaml`

用途：

- 專門輸出原生詞組碼表裡已有的詞。
- 不使用用戶詞典。
- 不動態調頻。
- 保留逐鍵提示，也就是你輸入未完成詞碼時，仍能看到補全提示。

例如原生詞組：

```text
你們    kckej
```

現在輸入 `kcke` 時，仍可看到「你們 7-」這類提示；但選過多少次都不會改變原生詞組碼表本身的排序。

核心設定在 `array30.custom.yaml`：

```yaml
fixed_phrases:
  dictionary: array30_phrases
  enable_completion: true
  enable_sentence: false
  enable_user_dict: false
  initial_quality: 900
```

這裡的重點是：

- `dictionary: array30_phrases`：只看原生詞組詞典。
- `enable_completion: true`：恢復原生詞組的逐鍵補全提示。
- `enable_user_dict: false`：原生詞組不參與用戶調頻。
- `initial_quality: 900`：讓原生詞組固定候選有足夠高的優先度，但仍低於單字固定層。

### C：動態連打學習櫃台

負責來源：

- 新建的 `array30_dynamic.dict.yaml`

用途：

- 專門負責「連續打單字」形成詞組或句子。
- 保留用戶詞典。
- 保留語句連打。
- 保留自動造詞與提交歷史學習。

例如：

```text
習    bblp
慣    nszm
```

你連續輸入：

```text
bblpnszm
```

如果第一次 Rime 組出「習實」，你手動選成「習慣」。多選幾次之後，C 這條動態管線會記住：

```text
習慣    bblpnszm
```

下次再輸入 `bblpnszm`，就會優先出「習慣」。

核心設定在 `array30.custom.yaml`：

```yaml
translator:
  dictionary: array30_dynamic
  enable_completion: true
  enable_encoder: true
  enable_sentence: true
  enable_user_dict: true
  encode_commit_history: true
  max_phrase_length: 5
  user_dict: array30
```

這裡的重點是：

- `dictionary: array30_dynamic`：主翻譯器不再直接吃完整的 `array30`，而改吃新的動態詞典。
- `enable_user_dict: true`：保留學習能力。
- `enable_sentence: true`：允許連續打字組句。
- `enable_encoder: true`：允許 Rime 把你選出的詞編成碼。
- `encode_commit_history: true`：把提交過的內容寫入學習歷史。
- `user_dict: array30`：沿用原來的 `array30.userdb`，避免換一個全新的用戶詞典導致以前資料全斷掉。

## 三、新建了哪些文件

### 1. `array30_char.schema.yaml`

作用：

讓 Rime 部署時能編譯 `array30_char.dict.yaml`。

這個 schema 不是給你切換使用的輸入方案，而是輔助編譯用的。可以把它想成一張「編譯通行證」：有了它，Rime 才會把單字詞典編成可載入的二進制資料。

關鍵設定：

```yaml
translator:
  dictionary: array30_char
  prism: array30_char
  enable_completion: false
  enable_sentence: false
  enable_user_dict: false
```

### 2. `array30_phrases.schema.yaml`

作用：

讓 Rime 部署時能編譯 `array30_phrases.dict.yaml`，並讓固定詞組層使用。

關鍵設定：

```yaml
translator:
  dictionary: array30_phrases
  prism: array30_phrases
  enable_completion: true
  enable_sentence: false
  enable_user_dict: false
```

其中 `enable_completion: true` 是後來補上的，目的是恢復你以前習慣的詞組補全提示，例如輸入 `kcke` 時看到「你們 7-」。

### 3. `array30_dynamic.dict.yaml`

作用：

這是專門給 C 使用的動態學習詞典。

它故意不引入 `array30_phrases`，只引入：

```yaml
import_tables:
  - array30_main
  - array30_emoji
```

原因是：原生詞組已經交給 B 的 `fixed_phrases` 固定層處理。如果 C 也吃 `array30_phrases`，那原生詞組又會重新進入用戶詞典調頻，B 就不乾淨了。

這個文件裡還加入了 encoder 規則：

```yaml
encoder:
  exclude_patterns:
    - '^.*!$'
    - '^.*@$'
  rules:
    - length_equal: 2
      formula: "AaAbAcAdBaBbBcBd"
    - length_equal: 3
      formula: "AaAbAcAdBaBbBcBdCaCbCcCd"
    - length_equal: 4
      formula: "AaAbAcAdBaBbBcBdCaCbCcCdDaDbDcDd"
    - length_equal: 5
      formula: "AaAbAcAdBaBbBcBdCaCbCcCdDaDbDcDdEaEbEcEd"
```

這組規則的意思是：C 的自動造詞採用「單字全碼串接」。

例如：

```text
習    bblp
慣    nszm
```

自動學到的詞碼就是：

```text
習慣    bblpnszm
```

這一點非常重要。早期我們曾使用行列詞組取碼方式，會把「習慣」學成 `bpnm`，而你實際輸入的是 `bblpnszm`，所以命不中。改成全碼串接後，C 的學習結果才會和你的連打方式一致。

### 4. `array30_dynamic.schema.yaml`

作用：

讓 Rime 部署時能編譯 `array30_dynamic.dict.yaml`。

關鍵設定：

```yaml
translator:
  dictionary: array30_dynamic
  prism: array30_dynamic
  enable_completion: true
  enable_encoder: true
  enable_sentence: true
  enable_user_dict: true
  encode_commit_history: true
  max_phrase_length: 5
  user_dict: array30
```

## 四、改動了哪些原有文件

### 1. `array30.custom.yaml`

這是本次最核心的配置文件。

主要改動包括：

1. 新增 `character`，作為 A 的固定單字管線。
2. 新增 `fixed_phrases`，作為 B 的固定原生詞組管線。
3. 修改 `engine/translators`，讓三條管線都掛到引擎上。
4. 修改主 `translator`，讓它使用 `array30_dynamic`，專門處理 C。
5. 在 `schema/dependencies` 裡加入新輔助 schema，讓部署時能編譯它們。

現在 translators 的順序大致是：

```yaml
engine:
  translators:
    - punct_translator
    - table_translator@character
    - table_translator@fixed_phrases
    - table_translator
    - table_translator@wsymbols
    - table_translator@array30_query
    - script_translator@luna_pinyin
```

這個順序可以理解為：

1. 先處理標點。
2. 再給固定單字候選。
3. 再給固定原生詞組候選。
4. 最後讓主翻譯器處理動態連打學習。

### 2. `array30_dynamic.dict.yaml`

這是新建文件，但它承擔了原本主 `translator` 的動態學習基礎詞典角色。

它最重要的設計是：

- 不 import `array30_phrases`。
- 有 encoder 規則。
- 用全碼串接方式學習連打詞。

### 3. build 目錄

部署後，Rime 生成了以下二進制與 schema 文件：

```text
build/array30_char.*
build/array30_phrases.*
build/array30_dynamic.*
build/array30.schema.yaml
```

這些是 Rime 實際運行時讀取的生成物。源頭仍然是上面那些 `.yaml` 文件。

## 五、為什麼不能只靠一個 translator

如果只用一個 translator，就會遇到原始矛盾：

```text
開 user_dict：
  C 可以學習，但 A/B 也會調頻。

關 user_dict：
  A/B 固定了，但 C 不能學習。
```

這就像把「固定價目表」和「會員偏好推薦」放進同一套排序系統裡。你想讓價目表永遠固定，但又想讓會員推薦變聰明；如果不拆開，兩者就會互相影響。

所以這次的核心不是調一個開關，而是把責任分開：

```text
A 單字固定：character
B 原生詞組固定：fixed_phrases
C 連打學習：translator + array30_dynamic
```

## 六、最終效果

目前最終效果如下。

### A：單字不調頻

例如 `nor`：

```text
產
性
```

即使你多次選「性」，排序仍不會被動態調頻改變。

### B：原生詞組不調頻

例如原生詞組碼表中的詞，不會因為使用次數被重新排序。

同時，由於 `fixed_phrases/enable_completion: true`，原生詞組仍保留逐鍵提示能力。

例如：

```text
kcke
```

可以提前提示：

```text
你們    7-
```

### C：連打單字仍能學習

例如：

```text
習    bblp
慣    nszm
```

你輸入：

```text
bblpnszm
```

第一次可能出「習實」。你手動選「習」再選「慣」，打出「習慣」。重複幾次後，Rime 會把「習慣」記為這條全碼連打路徑的常用詞，之後會優先出「習慣」。

## 七、目前要注意的地方

1. C 的學習是寫入 `array30.userdb`。

   這是 Rime 的用戶學習資料庫，不是普通暫存檔。它記錄你的使用習慣。

2. 舊規則曾把「習慣」學成 `bpnm`。

   `bpnm` 接近原生詞組碼 `bpnmj` 的主體部分，但你用的是全碼連打 `bblpnszm`。兩條路徑不同，所以不會命中。現在已改成全碼串接。

3. A 和 B 的固定候選不會因學習而變動。

   這正是本次設計目的。C 會學習，但學習結果不應該反過來改動 A/B 的原始碼表排序。

4. 如果將來想改變 C 最多學幾字詞，可以調整 `max_phrase_length: 5`。

   目前最多學五字組合，這是參考 Rime 內建倉頡方案的保守設定。

## 八、一句話總結

這次改造的本質，是把原本一條混合管線拆成三條專職管線：

```text
固定單字    -> character      -> 不調頻
固定詞組    -> fixed_phrases  -> 不調頻，但保留補全
連打學習    -> array30_dynamic -> 可學習，按全碼串接記住你的用詞
```

這樣就同時達成了：

```text
A 單字嚴格不調頻
B 原生詞組嚴格不調頻
C 連續打單字仍能學習你的習慣
```
